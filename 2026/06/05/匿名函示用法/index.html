匿名函式用法

在ai寫程式的時代，基本功還是不能廢掉。以前對於匿名函式的用法其實知道怎麼用，但自己不會寫來用，要我解釋也說不出個所以然。直到今天實際派上用場了，突然有種開竅的感覺

我對他的簡單說明就是，可以用在方法裡面的方法。
一般大眾寫法可能是這樣

使用匿名函式後，是不是看起來更簡潔了。

還可以再更簡潔，使用區域函式(local function)

使用匿名函式的優點
  • 不用另外寫一個實體 Method（不用污染類別結構）
    如果我們為了滿足 Checkmarx 或是做一個簡單的 Trim()，就得在 Class 裡面宣告一個 private string CleanMyData(string data)，久而久之，你的類別裡就會塞滿一堆「只被呼叫過一次」的雞肋方法，程式碼變得很雜亂。
    使用 Lambda 運算式，你可以直接在原本的方法內部「就地（In-place）解決」，用完即丟，非常乾淨落。
  • 它是標準的「區域方法（Local Function/Closure）」
    在程式設計裡有一個專有名詞叫 區域函式（Local Function） 或 閉包（Closure）。它被宣告在某個方法內部，所以它的生命週期只存在於這個方法內。外面的其他程式碼完全看不到它、也無法呼叫它，具有極高的封裝性（非常安全，不用擔心別人誤叫）。
