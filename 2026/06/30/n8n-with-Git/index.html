另外特別說明一下這行，冒號左邊 ./n8n-workflows-backup 是主機(WSL)端的路徑,右邊 /backup/workflows 是container 內部的路徑。所以更準確的註解應該寫成「WSL 主機路徑 : Container 內路徑」。

Docker 在掛載 bind mount 時,如果 WSL 主機端 ~/n8n_local/n8n-workflows-backup 資料夾不存在,Docker 會自動建立它,所以理論上你什麼都不做,執行 docker compose up -d 之後它就會自動生成。
