##### 今天再研究MCP時，遇到了command屬性有時用node，有時又用npx。

---
* ##### node 就像是你的「微波爐」（執行環境）：它是你家廚房裡常駐的電器。你想吃微波食品（執行 JavaScript 程式碼），就必須把食物放進這台微波爐裡加熱。

* ##### npx 就像是「外送平台 / 臨時租借服務」：你突然想吃烤鴨，但你家沒有烤鴨機，你也不想特地買一台佔空間。你就用外送叫一次，吃完把垃圾丟掉，家裡廚房依然乾乾淨淨。


比較項目	node	npx
本質	負責執行「本機硬碟裡已經存在」的 JS 檔案。	負責「臨時從網上下載、執行、執行完就刪除」工具。
使用前提	你的電腦裡必須先有那個實體 .js 檔案（如你的 index.js）。	你的電腦不需要下載該專案，只要有網路就能憑空執行。
優點	
極速、穩定。


因為檔案就在硬碟裡，不需要網路。

省空間、免維護。


永遠保證用的是網路上最新版本。

缺點	如果路徑寫錯，或者沒先安裝，就會噴錯。	每次啟動都要聯網抓取，在沙盒環境（如 Claude）容易卡死。
