##### 弱點掃描修正一直是件痛苦的事情，但現在有了AI讓這件事情變簡單了許多。
<!-- more -->
#
#
#
---
##### 弱點掃描完成後，一定會有份掃描報告。可能是一份1.200頁的的pdf檔。以[SQL Injection]這個弱點當範例，可以先把[SQL Injection]相關報告的頁數，另存成一份pdf檔。並對Claude Code執行Prompt：
``` text
請根據弱點掃描報告：./202606弱掃/黑箱/SQL_Injection/SQL_Injection.pdf中的3項弱點進行弱點修正，並產出一份修正報告在相同目錄下
```
#
#
#
---
##### 接著就等AI依據pdf的內容進行程式修正。修正完成後，會依據我們的需求產出了一份修改報告，裡面包含了弱點說明，以及改善方法。
#
#
#
---
##### 等所有的弱點修改完之後，我們要把修復的這些弱點加入到我們的SECURITY.md裡面，當作Claude Code日後再產出程式時的資安規範，Prompt如下：
``` text
你現在是一名精通 Web 安全防禦與 .NET 體系架構的資深資安專家。
我希望利用手邊的「弱點掃描（AppScan）修正後的報告」，在專案中建立一套自動化的資安制約機制。請幫我同時建立/修改兩個檔案，並將資安編程規範模組化抽離。
請執行以下任務：

1. 在專案根目錄建立一個全新的 `SECURITY.md` 檔案：
   - 分析我提供的弱點報告，將其提煉成日常開發時必須死守的白名單與防禦規範。
   - 包含但不限於：Null Byte (空字元) 攔截機制、500 錯誤與 Stack Trace 遮蔽、檔案上傳的防空與大小限制、Cookie (SessionId) 安全配置等。
   - 每一條規範都必須附上極度具體的 C# 程式碼或 XML 設定檔範例，並明確標註【❌ 錯誤範例（會被 AppScan 抓到）】與【⭕ 正確範例（符合資安規範）】。

2. 在專案根目錄建立（或更新）`CLAUDE.md` 檔案：
   - 除了保留專案原本的編譯與測試指令外，新增一個核心區段：`## 🚨 CRITICAL: Security & Compliance`。
   - 在該區段內明確寫出：你在進行任何程式碼變更、重構或新功能開發前，【必須】優先讀取並完全遵循 `./SECURITY.md` 中的資安規範。任何寫出符合錯誤範例特徵、或是漏掉防禦邏輯的程式碼，都將被視為重大 Bug。

以下是我的弱點掃描修正報告的路徑內容：
--------------------------------------------------
./202606弱掃/黑箱/*/*_Fix_Report.md
--------------------------------------------------
```
#
#
#
---
##### 之後也是用一樣的方式，慢慢的把SECURITY.md裡面的資安規範慢慢更新補齊。
#
#
#
---
##### 最後，還是要提醒自己，AI修改後的程式一定要在自己review過一次，除了code review外，也能藉此來增加自己對於資安方面的知識，而不是全部交由AI處理後，自己像船過水無痕般的一點進步都沒有。
